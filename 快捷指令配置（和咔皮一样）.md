# 快捷指令配置（和咔皮记账一样）

## 🎯 目标效果

和咔皮记账一样：**截屏 → 从截屏获取图像 → 打开 bill**

## ✅ 推荐配置（3步搞定）

### 方案1: 使用URL + Base64（最像咔皮）⭐️

```
步骤1: 截屏
步骤2: 从 截屏 获取图像
步骤3: 编码
       - 编码方式: Base64
       - 输入: 图像
步骤4: 打开URL
       - URL: bill://
       - 在URL后面点击"+"号
       - 选择: Base64编码的结果
```

**最终URL格式**: `bill://[Base64编码的图像数据]`

### 方案2: 使用URL参数

```
步骤1: 截屏
步骤2: 从 截屏 获取图像
步骤3: 编码
       - 编码方式: Base64
       - 输入: 图像
步骤4: 打开URL
       - URL: bill://?data=
       - 在?data=后面点击"+"号
       - 选择: Base64编码的结果
```

**最终URL格式**: `bill://?data=[Base64编码的图像数据]`

### 方案3: 临时使用剪贴板（最简单）

```
步骤1: 截屏
步骤2: 从 截屏 获取图像
步骤3: 拷贝到剪贴板
       - 输入: 图像
步骤4: 打开 bill
```

## 📱 详细配置步骤（方案1）

### 第1步：添加"截屏"操作
1. 打开快捷指令App
2. 点击右上角"+"创建新快捷指令
3. 搜索"截屏"
4. 选择"截屏"操作
5. 保持默认设置

### 第2步：添加"从截屏获取图像"
1. 点击搜索框
2. 搜索"获取图像"
3. 选择"从XX获取图像"
4. 确保输入是"截屏"

### 第3步：添加"编码"
1. 搜索"编码"或"Base64"
2. 选择"编码"操作
3. 编码方式选择：**Base64**
4. 输入选择：**图像**

### 第4步：添加"打开URL"
1. 搜索"打开URL"
2. 选择"打开URL"操作
3. 在URL输入框输入：`bill://`
4. **关键步骤**：
   - 点击`bill://`后面的位置
   - 点击右侧的变量按钮（或"+"号）
   - 选择"Base64编码的结果"
5. 最终显示应该是：`bill://` + `Base64编码的结果`

### 第5步：保存并命名
1. 点击右上角"完成"
2. 给快捷指令起个名字，比如："识别账单"
3. 设置图标（可选）

## 🔍 工作原理

### 方案1的工作流程：
```
截屏
  ↓
提取图像
  ↓
转Base64编码
  ↓
打开 bill://[Base64数据]
  ↓
App解析URL中的图片数据
  ↓
自动打开解析页面
  ↓
显示图片
```

### App支持的URL格式：
```
bill://                              → 打开App（检查剪贴板）
bill://[base64图片]                  → 直接传递图片
bill://?data=[base64图片]            → 通过data参数传递
bill://?image=[base64图片]           → 通过image参数传递
```

## 🎨 为什么咔皮不需要"拷贝到剪贴板"？

咔皮记账使用了以下技术：

1. **URL Scheme传递数据**
   - 快捷指令的"打开"操作实际上是打开URL
   - 可以在URL中附加数据（比如Base64编码的图片）
   - App解析URL参数获取图片

2. **我们的实现**
   - 支持URL路径传递：`bill://[base64]`
   - 支持URL参数传递：`bill://?data=[base64]`
   - 支持剪贴板检测：自动后备方案
   - 多重保险，确保能获取到图片！

## 💡 调试技巧

### 如果图片没有自动加载：

1. **检查快捷指令配置**
   - 确保"编码"步骤的输入是"图像"
   - 确保"打开URL"中正确拼接了Base64数据

2. **查看App日志**
   - 在Xcode中运行App
   - 查看Console输出
   - 会显示：
     ```
     📱 Received URL: bill://...
     📱 URL Scheme: bill
     ✅ Found image in URL path
     ```

3. **使用剪贴板方案作为备选**
   - 如果URL方案不工作
   - 可以临时使用剪贴板方案
   - App会自动检测剪贴板

## ⚠️ 注意事项

### URL长度限制
- iOS对URL长度有限制（通常2048字符）
- Base64编码后的图片可能很大
- 如果图片太大，建议使用剪贴板方案

### 推荐做法
1. **小图片**（<100KB）：使用URL方案（方案1/2）
2. **大图片**（>100KB）：使用剪贴板方案（方案3）
3. **不确定**：使用剪贴板方案，最稳定

| 方案 | 优点 | 缺点 | 推荐指数 |
|------|------|------|----------|
