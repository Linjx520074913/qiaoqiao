# 多账单识别功能说明

## 🎯 功能概述

现在App支持**一次识别多条账单**！

适用场景：
- 📱 微信/支付宝账单截图（多条交易记录）
- 🧾 购物小票（多个商品）
- 📊 银行流水截图
- 📝 手写账本照片

## ✨ 核心功能

### 1. 智能检测多条交易

App会自动检测图片中的**所有金额**：
- 识别每一行的金额
- 自动提取商户名称
- 智能分类每笔交易
- 显示交易数量

## 🔍 识别逻辑

### 检测流程

1. **OCR文字识别**
   - 提取图片中的所有文字
   - 按行分割

2. **查找所有金额**
   - 扫描每一行
   - 识别金额模式：
     - ¥123.45
     - 123.45元
     - $123.45
     - -123.45（支出）
     - +123.45（收入）

3. **提取商户信息**
   - 移除金额后的剩余文字
   - 从上下文推断商户名
   - 清理特殊字符

4. **智能分类**
   - 根据关键词自动分类
   - 8大分类：餐饮、购物、交通等

5. **生成账单列表**
   - 每个金额一条账单
   - 按顺序编号
   - 显示详细信息

## 🎨 智能识别规则

### 金额识别

**支持格式**：
- `¥35.00` → 35.00
- `35.00元` → 35.00
- `$35.00` → 35.00
- `-35.00` → 35.00（支出）
- `+8500.00` → 8500.00（收入）

**过滤规则**：
- 金额 > 0
- 金额 < 1,000,000
- 自动去除异常值

### 商户识别

**提取逻辑**：
1. 移除金额部分
2. 清理空白字符
3. 如果太短，从上下文提取
4. 限制最长30字符

**示例**：
```
"星巴克咖啡 ¥35.00" → "星巴克咖啡"
"¥6.00 地铁" → "地铁"
"超市购物 128.50元" → "超市购物"
```

### 分类识别

**关键词匹配**：

| 分类 | 关键词 |
|------|--------|
| 🍽️ 餐饮 | 餐、咖啡、美食、饭店、外卖、starbucks |
| 🛒 购物 | 超市、商场、购物 |
| 🚗 交通 | 地铁、出租、滴滴、uber、交通、打车 |
| 🎮 娱乐 | 电影、娱乐、ktv |
| 🏠 住房 | 房租、物业、水电 |
| 💗 医疗 | 医院、药店、医疗 |
| 📚 教育 | 教育、培训、学费 |
| ⚪ 其他 | 未匹配 |

## 💡 使用技巧

### 1. 拍摄清晰的图片

✅ **推荐**：
- 光线充足
- 文字清晰
- 对焦准确
- 避免倾斜

❌ **避免**：
- 模糊不清
- 反光严重
- 文字太小
- 角度太斜

### 2. 格式建议

**最佳格式**（每行一条）：
```
星巴克 ¥35.00
地铁 ¥6.00
超市 ¥128.50
```

**也支持**：
```
¥35.00 星巴克
6.00元 地铁出行
超市购物 128.50
```

### 3. 检查识别结果

**OCR识别结果区域**：
- 查看提取的原始文字
- 确认是否完整
- 检查是否有遗漏

**账单信息区域**：
- 检查账单数量
- 验证金额准确性
- 确认分类正确

### 4. 修正识别错误

如果识别有误：
1. 查看OCR原始文字
2. 检查图片质量
3. 重新拍摄清晰照片
4. 或手动输入

## 🐛 常见问题

### Q: 为什么只识别到1条账单？

**可能原因**：
1. 图片中只有一个金额
2. 其他金额格式不标准
3. OCR未识别到文字

**解决方案**：
- 查看OCR识别结果
- 确认文字是否清晰
- 重新拍摄

### Q: 识别的商户名不对？

**原因**：文字识别错误或格式特殊

**解决方案**：
- 查看OCR原始文字
- 确认识别是否准确
- 手动编辑（保存后）

### Q: 金额识别错误？

**检查**：
- OCR是否识别准确
- 金额格式是否支持
- 是否有特殊符号干扰

**解决方案**：
- 重新拍摄清晰照片
- 手动修正

### Q: 分类不准确？

**原因**：关键词匹配失败

**解决方案**：
- 保存后手动修改分类
- 或在商户名中添加关键词
